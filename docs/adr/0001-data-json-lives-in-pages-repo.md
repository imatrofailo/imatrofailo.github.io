# 0001 — JSON index files live in the Pages repo, not the wiki repo

The public site (imatrofailo.github.io) needs `data/topics.json` and `data/tips.json` to render bubbles, charts, and practical tips. These files are generated by `imatrof-wiki/scripts/generate_index.py` from raw Telegram and YouTube sources.

**Decision:** generated JSON files are committed directly to this repo (`imatrofailo.github.io/data/`), not to `imatrof-wiki`.

**Why:** this repo is the single source of truth for everything the public site serves. Keeping data here means a simple `git push` deploys both HTML and data atomically. The alternative — fetching raw GitHub URLs from the wiki repo — would couple the public site to the wiki repo's branch structure and access model (wiki repo may become private).

**Trade-off accepted:** data files are derived artifacts committed to git. They must be regenerated and re-pushed whenever raw sources update. This is intentional — done manually via the `wiki-deploy` skill in imatrof-marketplace.
